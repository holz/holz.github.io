如何在脚本中初始化 Surfer 实例

  • Script
  • Surfer

要用脚本完成一些 Surfer 自动化任务，首先要解决的是，如何获取一个实例。

Surfer 的官方帮助给出的多是简单粗暴的新建实例方法，若干脚本运行下来，不，只要一个脚本运行个若干次，屏幕上就多出一堆 Surfer 窗口。

实际上，已经运行的 Surfer 实例可以通过 GetObject() 函数获取。所以在做 Surfer 自动化时，可以先从内存在获取实例，内存中没有再新建一个。

完成自动化任务后，对 Surfer 实例的处理也分两种情况，自己新建的直接关闭，否则保留。如此就需要一个判断标记。

怎么应用？假设写的脚本要处理某些数据文件，那么初始化 Surfer 之后就该读数据了。一般我在这里做程序终止判断，假如选择文件的时候按了“取消”，就结束。

以上搞这么多复杂的辅助代码，都是因为我自己电脑配置差，内存吃紧，要节约每一个字节。要是像现在标配 8G 内存，动不动几十个 GB 内存，哪里要这么复杂啊。初始化只要一句 Set wsApp=CreateObject("Surfer.Application") 就可以了，完事也不用判断来判读去，直接 wsApp.Quit 解决。
