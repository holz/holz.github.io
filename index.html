只网格化有数据区域

Surfer 在网格化的时候，是将离散的数据变成一个规则的矩形网格，没有数据的地方，Surfer 会通过内插或外插的方式填充数据。

在实际应用中，我们往往希望那些没有数据的区域保持空白。

传统做法是先做网格化，再通过数字化或其他方式建立边界文件，然后进行白化。这样的做法手续繁杂，遇到大量数据时简直烦不胜烦。

若干年前，只想着怎么自动生成边界文件，于是到 Golden Software 论坛寻求帮助， 幸得一众高手接连出招。

选了其中简单的方法，用了几年，效果算不错。

原理很简单，首先正常网格化，用你喜欢的网格化方法和参数，得到网格 A。

再用合适的搜索半径配合 Data Metrics 方法建立一个辅助网格，Grid -> Data -> Data Metrics -> Advanced Options -> Count -> Search Radii ，暂且称为 Aux 网格。

然后在 Aux 网格基础上建立遮罩网格，把 Aux 网格中所有小于 1 的值替换为白化值（Surfer 默认是 1.70141e+038），其他值全部替换为 0。使用 Grid -> Math 命令配合公式 IF(A <1,1.70141e+038,0) 即可得到网格 B。

用 Grid -> Math 命令配合公式 A + B， 让网格 A 和网格 B 相加得到最终结果，且叫它网格 C。

如何在脚本中初始化 Surfer 实例

要用脚本完成一些 Surfer 自动化任务，首先要解决的是，如何获取一个实例。

Surfer 的官方帮助给出的多是简单粗暴的新建实例方法，若干脚本运行下来，不，只要一个脚本运行个若干次，屏幕上就多出一堆 Surfer 窗口。

实际上，已经运行的 Surfer 实例可以通过 GetObject() 函数获取。所以在做 Surfer 自动化时，可以先从内存在获取实例，内存中没有再新建一个。
